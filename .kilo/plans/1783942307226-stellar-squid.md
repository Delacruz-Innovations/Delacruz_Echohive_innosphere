## Component structure & layout
`<section id="hero" className="relative ...">` containing:
- A `grid lg:grid-cols-2` wrapper.
- **Left column** (DOM order): eyebrow (`<p>`, uppercase tracking, with `eyebrowSub` smaller line) → `<h1>` headline → subheadline (`<p>`) → CTA row (primary + secondary) → *(trust statement placed after the dashboard on mobile, see below)*.
- **Right column**: `BusinessPerformanceDashboard` card.
- **Trust statement**: rendered as a third grid child with `lg:col-start-1` so on `lg` it sits under the CTAs in column 1, and on mobile/tablet (single column) it flows after the dashboard — satisfying:
  - Desktop: two columns (left text, right dashboard).
  - Tablet (`md`, <`lg`): single-column stack with dashboard beneath CTAs.
  - Mobile: Headline → Copy → CTA → Dashboard → Trust Statement.

### CTAs
- **Primary** → `<CalendlyPopup text={heroContent.primaryCTA.label} className="…" />` (opens the existing 30-min Calendly popup). Render with a clear focus-visible ring and `aria-label`.
- **Secondary** → `<Link to={heroContent.secondaryCTA.to}>` styled button.
- **Sticky tertiary "Speak to an Expert"** → `fixed bottom-6 right-6 z-40` `<CalendlyPopup>` with distinct styling; only rendered while the hero is mounted (homepage).

### Scroll indicator
- A bottom-centered control labeled `heroContent.scrollLabel` with a downward chevron and `aria-hidden` decorative arrow.
- `onClick`: `heroRef.current.nextElementSibling?.scrollIntoView({ behavior: reducedMotion ? 'auto' : 'smooth' })` (scrolls to the next homepage section, currently `Casestudy`).

## Business Performance Dashboard
- Card with heading **"Business Performance Snapshot"** and the 6 `dashboardMetrics`.
- Each metric row: label + illustrative value with a count-up animation + `↑`/`↓` indicator (green for up, red/amber for down). Decorative arrows `aria-hidden`.
- **Count-up**: `useEffect` + `requestAnimationFrame` tween 0→`value` when the card enters the viewport (IntersectionObserver). If `prefers-reduced-motion`, set final value immediately (no animation).
- Footer note = `dashboardNote` (illustrative disclaimer).
- Soft hover lift on metric rows (`hover:-translate-y-0.5` / shadow), disabled under `motion-reduce`.

## Accessibility (WCAG 2.2 AA)
- **Single `<h1>`**: hero headline is the only `<h1>`; navbar logo `<h1>` → `<span>` (Navbar edit).
- Semantic hierarchy: eyebrow/subheadline/trust as `<p>`; dashboard heading as `<h2>` if needed.
- Colour contrast: white/light text on the existing dark background; ensure CTA buttons meet AA contrast.
- Keyboard-accessible CTAs: real `<button>`/`<a>` with visible `focus-visible` ring.
- Screen-reader labels: `aria-label` on CTAs and the scroll control; `aria-hidden` on decorative arrows/chevrons.
- Reduced motion: Tailwind `motion-reduce:` utilities + JS `prefers-reduced-motion` check disable count-up, fade-ins, and smooth-scroll.
- Responsive typography/spacing via Tailwind breakpoints.

## Animation

- Soft hover on dashboard rows.
