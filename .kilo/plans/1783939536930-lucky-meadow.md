# SEO Audit & Implementation Plan — Delacuz Innovations

## Executive Summary
This is a React 19 + Vite + React Router 7 SPA with **no per-page metadata, no structured data, no robots.txt, no sitemap, and broken internal links**. Google indexes the static shell in `index.html` and largely ignores the client-rendered content. The fixes below will make the site crawlable, indexable, and richer for search.

---

## Phase 1: Critical SEO Infrastructure (Highest Impact)

### 1.1 Install `react-helmet-async` for per-page head management
**File:** `package.json`
- Add `react-helmet-async` dependency
**Why:** Without this, every page shares the same static `<title>` and has no meta description, canonical, or OG tags. This is the single biggest SEO blocker.

### 1.2 Wrap app in `HelmetProvider`
**File:** `src/App.jsx`
- Import `{ Helmet, HelmetProvider }` from `react-helmet-async`
- Wrap `<Router>` with `<HelmetProvider>`
**Why:** Enables all child routes/components to inject `<title>`, `<meta>`, `<link rel="canonical">`, and JSON-LD scripts.

### 1.3 Add `public/robots.txt`
**File:** `public/robots.txt` (new)
```
User-agent: *
Allow: /
Disallow: /offices/
Sitemap: https://delacruzinnovation.com/sitemap.xml
```
**Why:** Tells crawlers what to index and where to find the sitemap. `/offices/` is largely location data already visible on `/contact`.

### 1.4 Add `public/sitemap.xml`
**File:** `public/sitemap.xml` (new)
- Static XML sitemap listing all public routes with `lastmod` dates
**Why:** Accelerates discovery of service pages, insights, case studies, and job listings.

## Phase 2: Per-Page Metadata & Structured Data

### 2.1 Create a reusable `SEO` component
**File:** `src/utils/SEO.jsx` (new)
- Accepts `title`, `description`, `canonical`, `ogImage`, `ogType`, `jsonLd`
- Renders `<Helmet>` with:
  - `<title>` (with brand suffix)
  - `<meta name="description">`
  - `<link rel="canonical">`
  - Open Graph tags (title, description, image, url, type)
  - Twitter Card tags
  - JSON-LD `<script type="application/ld+json">`
- Includes base Organization schema shared across all pages

### 2.2 Add Organization + WebSite + SearchAction schema (site-wide)
**File:** `src/utils/SEO.jsx`
- `Organization`: name, url, logo, sameAs (social profiles), contactPoint

**Why:** Enables Google Knowledge Graph, entity recognition, and site-links search box.

## Phase 3: Content & Internal Linking Fixes

### 3.1 Fix broken service links in HelpOurClient
**File:** `src/Components/HelpOurClient.jsx`
- Change `/services/technology-it-services` → `/services/digital-transformation`
- Change `/services/cyber-security-consultation` → `/services/SaaS-PaaS-Development`
**Why:** Those slugs do not exist. Broken links hurt crawlability and user experience.

### 3.2 Add breadcrumb navigation component
**File:** `src/components/Breadcrumbs.jsx` (new)
- Auto-generates breadcrumbs from `location.pathname`
- Uses `BreadcrumbList` JSON-LD
- Renders `<nav aria-label="Breadcrumb">` with `<ol>` and `<li>`
- Mount in `App.jsx` above `<AppRoutes />` or in each page
**Why:** Breadcrumbs improve crawl paths and SERP appearance.

### 3.3 Improve Homepage H1 and copy
**File:** `src/Pages/Homepage.jsx` and `src/Components/Hero.jsx`
- Replace animated typing H1 with static, descriptive H1: "Digital Transformation, SaaS Development & IT Consulting for Nigeria & the UAE"
- Add a subheading that clearly states the value proposition and target markets
- Keep animation as a decorative element below the H1, not as the H1 itself
**Why:** Google needs a stable, crawlable H1. Animated/changing text is not reliably indexed.

### 3.4 Add service-specific CTAs and cross-links
- In `ServiceDetailsPage.jsx`, add "Related Services" section linking to other services
- In `Homepage.jsx`, ensure each major section links to relevant service detail pages
**Why:** Strengthens internal link equity flow to deep pages.

### 3.5 Fix generic alt text
- `Hero.jsx`: change `alt="Background"` to `alt="Delacuz Innovations digital transformation team"`
- Any other generic `alt=""` or `alt="image"` → descriptive
**Why:** Alt text supports image search and accessibility.

## Phase 4: Performance & Technical SEO

### 4.1 Lazy-load below-the-fold images
**File:** All pages with background images (`ContactPage`, `InsightDetailPage`, `InsightsListPage`)
- Add `loading="lazy"` to images below the fold (hero images can remain eager)
**Why:** Improves Core Web Vitals (LCP) without losing above-fold content.

### 4.2 Preload critical assets in `index.html`
- Add `<link rel="preload">` for logo and critical fonts if any
**Why:** Reduces LCP by 100-200ms.

### 4.3 Add `loading="lazy"` to iframe in `ContactPage`
- Already has `loading="lazy"` on Google Maps iframe — verify it stays.

### 4.4 Avoid render-blocking scripts
- Move GTM to `</body>` or use `defer`/`async` properly. Current inline script in `<head>` is render-blocking.
- Alternatively, keep GTM in head but ensure it's the only blocking script.

## Phase 5: Accessibility → SEO

### 5.1 Semantic landmarks
- Ensure `<main>`, `<header>`, `<nav>`, `<footer>` are present. Currently `<nav>` is inside a `<div>` in Navbar; add `<header>` and `<main>` landmarks in `App.jsx`.
**Why:** Screen readers and Google use landmarks to understand page structure.

### 5.2 Skip-to-content link
- Add in `App.jsx`: `<a href="#main-content" className="sr-only focus:not-sr-only">Skip to main content</a>`
**Why:** Accessibility is a ranking signal per Google guidelines.

### 5.3 Button/link semantics
- In `Navbar.jsx`, replace `<li>` inside `<nav>` with proper `<ul>` or use `<div>` with `role="list"`. Current `<li>` without parent `<ul>` is invalid HTML.
**Why:** Invalid HTML can confuse crawlers and assistive tech.

## Remaining Gaps (Post-Implementation)
1. **SSR/SSG migration** — For maximum SEO, consider migrating to Next.js. Until then, the SPA will always be at a crawlability disadvantage versus pre-rendered sites.
2. **Dynamic sitemap** — If insights/jobs change frequently, generate sitemap dynamically from Firebase rather than maintaining static XML.
3. **Hreflang** — If targeting multiple regions/languages, add hreflang tags.
4. **AMP** — Not recommended unless news-heavy; skip for now.
5. **Structured data testing in production** — Monitor GSC for structured data errors after deployment.
